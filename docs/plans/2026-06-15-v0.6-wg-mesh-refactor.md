# wg-mimic-fabric v0.6 — WireGuard 组网 + Mimic 伪装 重构实现计划

> **For agentic workers:** 用 executing-plans 技能逐任务实现。步骤用 `- [ ]` 跟踪。这是 Bash 单文件工具，"测试" = `bash -n` 语法检查 + `shellcheck -S error` + `scripts/smoke.sh`（对纯函数做渲染/编解码断言，无需 root）。

**Goal:** 把 wg-mimic-fabric 重构为「公网入口 ⇄ IX 经 WireGuard 组网、Mimic 把 WG 的 UDP 伪装成 TCP、IX 经 nft 转发到落地」的中转工具，借鉴 ix-transit-fabric 的角色/转发/接入码/多规则模型。

**Architecture:** 单条 WireGuard 点对点隧道（入口 10.88.0.1 ⇄ IX 10.88.0.2），两端各跑 Mimic 把 WG 的 UDP 端口伪装为 TCP；IX 为 WG 监听端并用 nft 把 `IX虚拟IP:transit_port` DNAT 到落地；公网入口用 nft 把 `公网:client_port` DNAT 到 `IX虚拟IP:transit_port`（经 WG 路由）。多条转发规则复用同一条 WG 隧道，仅在两端各加一组 nft DNAT。接入码由 IX 生成、公网入口单向导入（含 WG 组网密钥、虚拟IP、规则数组、MTU）。

**Tech Stack:** Bash 5（单文件 `install.sh`，软链为 `wm`）、WireGuard(`wg`/`wg-quick`)、Mimic(hack3ric/mimic, eBPF/XDP, 内核≥6.1)、nftables、systemd、python3（仅 JSON/base64url）。

---

## 角色与数据流

```text
客户端
  → 公网入口 公网IP:client_port            (rule.client_port，nat-ingress)
  → 公网入口 nft DNAT → IX虚拟IP:transit_port
  → WireGuard 隧道（Mimic 伪TCP 承载 WG 的 UDP）
  → IX 10.88.0.2:transit_port               (nat-transit)
  → IX nft DNAT → 落地 LANDING_HOST:LANDING_PORT
```

- 角色 `nat-transit`：IX/落地侧。WG 监听端 + Mimic(filter local) + nft(转发到落地) + 生成接入码。
- 角色 `nat-ingress`：公网入口。导入接入码 + WG 连入(client) + Mimic(filter remote) + nft(客户端入口→IX虚拟IP:transit_port)。
- 一条 WG 隧道承载全部规则；规则差异只体现在两端 nft DNAT 与入口的 client_port。

### 2) rule env（规则级）`profiles/<id>/rules/<rule_id>.env`

```sh
RULE_ID=rule-main
RULE_NOTE=默认转发
RULE_ENABLED=true
TRANSIT_PORT=40000            # IX 虚拟IP 上监听/被 DNAT 的端口
LANDING_HOST=1.2.3.4         # 落地
LANDING_PORT=50000
FORWARD_PROTO=both           # tcp|udp|both
CLIENT_PORT=30000            # 仅 nat-ingress：客户端入口端口（导入时分配）
```

### 3) 接入码 `WMGF1:` + base64url(JSON)，`code_schema=5`

```json
{
  "version": 1,
  "code_schema": 5,
  "project": "wg-mimic-fabric",
  "role": "nat-transit-code",
  "profile_id": "ix-nat",
  "profile_name": "ix-nat",
  "wg_mesh_subnet": "10.88.0.0/24",
  "ix_wg_ip": "10.88.0.2",
  "ingress_wg_ip": "10.88.0.1",
  "ix_wg_pubkey": "<IX WG 公钥>",
  "ingress_wg_privkey_b64": "<入口 WG 私钥 base64url>",
  "ix_endpoint_host": "nat.example",
  "wg_port": 51820,
  "wg_mtu": 1420,
  "mimic_keepalive": "300:::",
  "forward_proto": "both",
  "rules": [
    {"rule_id":"rule-main","note":"默认转发","transit_port":40000,"landing_host":"1.2.3.4","landing_port":50000,"proto":"both"}
  ],
  "rules_b64": "<base64url(TSV: rule_id\\tnote\\ttransit_port\\tlanding_host\\tlanding_port\\tproto 每行一条)>",
  "created_at": "2026-..Z"
}
```

- `rules` 给人/工具看；`rules_b64` 给解析（避免 JSON 数组解析在纯 bash 的痛点，用 TSV）。两者都写，解析优先 `rules_b64`。
- 落地 IP/端口、transit_port、MTU 均在码内（满足"接入码包含转发端口/IP/MTU"）。

### 4) nft 表（`table inet wg_mimic_fabric`）

IX(`nat-transit`)：prerouting 把 `WG_IX_IP:transit_port` DNAT→`landing:port`；postrouting masquerade；forward accept；input 开放 WG_PORT(tcp+udp，Mimic 用)。
入口(`nat-ingress`)：prerouting 把 `client_port` DNAT→`WG_IX_IP:transit_port`；postrouting masquerade(经 WG 出)；forward accept；input 开放每条 `client_port`(tcp/udp 按 proto)。

### 5) systemd

- `wg-mimic-mimic@<iface>.service`（按网卡，Type=notify，`mimic run %i -F /etc/mimic/%i.conf`）
- `wg-mimic-tunnel@<profile>.service`（Type=oneshot，`wg-quick up /etc/wireguard/<WG_IFACE>.conf`）
- per-profile drop-in 把 tunnel 绑定到正确 `mimic@<iface>`（沿用已修复方案）
